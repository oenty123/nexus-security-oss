# Гайд: расширение Nexus Security для VS Code

Пошагово, для тех, кто только установил. English version below.

## 1. Первый запуск

После установки расширения откройте **папку проекта** (File → Open Folder), а не
отдельный файл. Если VS Code спросит про доверие к папке — нажмите **Trust**,
иначе анализ не запустится.

Расширение само найдёт Python и движок. Ничего настраивать не нужно — но если
оно что-то не нашло, см. раздел «Настройка» ниже.

## 2. Как увидеть проблемы

Откройте любой файл с кодом (`.py`, `.js`, `.ts`, `.php`, `.go` …). Анализ
запустится автоматически при открытии и при сохранении.

**Найденные проблемы подчёркиваются волнистой линией прямо в коде:**

- 🔴 красная волнистая линия — критичная или высокая уязвимость
- 🟡 жёлтая — средняя
- 🔵 синяя/серая — низкая или замечание по качеству

**Наведите курсор на подчёркнутую строку** — появится всплывающая подсказка
с описанием: что за уязвимость, чем опасна, и (если включено) её код правила
и ссылка на CWE. Это главный способ узнать, что не так.

## 3. Панель «Проблемы» (Problems)

Все находки сразу — внизу во вкладке **Problems** (View → Problems, или
`Ctrl+Shift+M`). Клик по строке — переход к месту в коде. Удобно, когда
проблем много и нужен общий список.

## 4. Статус-бар

Внизу справа — значок Nexus с оценкой файла и числом проблем
(например `🛡️ Nexus B (3)`). Клик по нему запускает повторный анализ.

## 5. Быстрые исправления (лампочка 💡)

Наведите курсор на подчёркнутую проблему и нажмите значок **лампочки** (или
`Ctrl+.`). Появится меню действий:

- **применить предложенное исправление** (если оно есть для этого правила)
- **подавить эту находку** — вставит комментарий `# nexus:ignore[ID]`
  (или `// nexus:ignore`, в зависимости от языка), и предупреждение исчезнет

## 6. Команды (палитра)

Нажмите `Ctrl+Shift+P` и наберите «Nexus» — доступны команды:

| Команда | Что делает |
|---------|------------|
| **Scan Current File** | пересканировать открытый файл |
| **Scan Entire Workspace** | проверить весь проект |
| **Объяснить код** | панель со структурным разбором: что делает каждая функция |
| **Добавить docstring-заготовки** | вставить шаблоны документации (Python) |
| **Оптимизировать и упростить код** | безопасный рефакторинг всего файла |
| **Упростить функцию** | рефакторинг функции под курсором (Python) |
| **Показать журнал находок** | сводный отчёт по последнему сканированию |
| **Clear All Warnings** | убрать всю подсветку |
| **Открыть настройки Nexus** | страница настроек |

Команда «Оптимизировать» доступна и по правому клику в редакторе.

## 7. Настройка (если нужно)

**Открыть настройки Nexus** (через палитру) или Settings → Extensions → Nexus.
Полезные опции:

- **minSeverity** — минимальный уровень для показа (скрыть мелочь: поставьте `medium`)
- **showAntipatterns** — показывать ли проблемы качества кода (магические числа,
  сложность). Выключите, если хотите только безопасность
- **runOnType** — анализировать при наборе (по умолчанию — при сохранении)
- **refactorLevel** — агрессивность рефакторинга: `1` безопасный (рекомендуется),
  `2` стандарт, `3` агрессивный
- **showRuleId / showCwe** — показывать `[PY-SSTI-001]` и `(CWE-89)` в подсказке
- **showStatusBar** — значок со счётчиком внизу
- **pythonPath / cliPath** — пути к Python и движку (заполняются авто-поиском;
  задайте вручную, только если не нашлись)

## 8. Если подсветки нет

1. Открыта ли **папка** (не один файл) и доверена ли она (Trust)?
2. Посмотрите **View → Output → Nexus Security** — там причина (нет Python,
   не найден `cli.py` и т.п.).
3. Запустите вручную: палитра → **Scan Current File**.

# Guide: Nexus Security VS Code extension (English)

## Quick use

1. Open a **project folder** and trust it if asked.
2. Open a code file — analysis runs on open and on save.
3. **Issues are underlined** in code: red = critical/high, yellow = medium,
   blue/grey = low. **Hover the underline** to read what's wrong and how to fix it.
4. See all issues in the **Problems** panel (`Ctrl+Shift+M`); click to jump.
5. **Lightbulb** (`Ctrl+.`) on an issue → apply a fix or suppress it
   (`# nexus:ignore[ID]`).
6. `Ctrl+Shift+P` → "Nexus" for commands: scan, explain code, add docstrings,
   optimize, show report.

## Settings worth knowing

`minSeverity` (hide noise), `showAntipatterns` (security-only vs +quality),
`runOnType` (live vs on-save), `refactorLevel` (1 = safest), `showRuleId`/`showCwe`
(extra info in tooltips). Paths auto-detect; set `pythonPath`/`cliPath` manually
only if detection fails.

## No highlights?

Open a folder (not a single file), trust it, check **Output → Nexus Security**
for the reason, or run **Scan Current File** manually.

## Без Python: использование с бинарником

Если у вас есть автономный бинарник `nexus` (из релиза на GitHub), расширение
использует его автоматически — Python не нужен. Расширение ищет бинарник:
рядом с проектом, в `bin/`, или в PATH. Можно указать путь вручную в настройке
**binaryPath**. Приоритет: бинарник → Python+cli.py.
